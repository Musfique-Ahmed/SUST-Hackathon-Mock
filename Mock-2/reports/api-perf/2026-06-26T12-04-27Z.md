## Top Issues

- Pinned `pydantic==2.9.2` cannot be installed on Python 3.14 hosts without a Rust toolchain. No prebuilt wheel. Should be loosened to `pydantic>=2.9,<3` in `requirements.txt` to allow a prebuilt 2.13.x fallback (zero app changes needed).
- T-1 README mismatch: the longer example body in the API section implies `human_review_required: true`, the shorter T-1 in the verification table implies `false`. Spec ambiguity — pick one canonical form and stick to it.
- Docker daemon was not running on the test host, so `docker compose up` was not viable. Server was run from a local venv instead. Not a service defect, just a host-setup gotcha.

## Recommendations

- Loosen `pydantic` pin in `F:\Git Repo\SUST-Hackathon-Mock\ticket-sorter\requirements.txt` to `pydantic>=2.9,<3` to make `pip install -r requirements.txt` work on Python 3.12+ hosts without a Rust compiler.
- Decide which T-1 wording is the contract (short form = `human_review_required:false`, or the longer API example which arguably deserves `true`), and update README so the two tables agree.
- Add a `pytest` harness around the 5 README cases so the verification table is enforced in CI, not just docs.
- Latency is fine (sub-30ms p99 in-process, single worker). The single-process uvicorn setup is the real ceiling — if sustained >2k req/s is needed, run with `--workers N` (4-8 typically suffices on this footprint).
- No rate limiting / auth is in scope for this service; document that explicitly so callers know not to expect it.

## Reproduce

```bash
# 1) Start the server (local venv path; Docker also works if daemon is up)
cd "F:\Git Repo\SUST-Hackathon-Mock\ticket-sorter"
python -m venv .venv
.venv\Scripts\pip install -r requirements.txt
.venv\Scripts\uvicorn app.main:app --host 0.0.0.0 --port 8000

# 2) Verify liveness
curl -s http://localhost:8000/health
# -> {"status":"ok"}

# 3) Smoke the 5 README cases + schema-fail + safety case
for body in \
  '{"ticket_id":"T-1","message":"I sent 3000 to wrong number"}' \
  '{"ticket_id":"T-2","message":"Payment failed but balance deducted"}' \
  '{"ticket_id":"T-3","message":"Someone called asking my OTP, is that bKash?"}' \

  echo "----- $body"
  curl -s -X POST http://localhost:8000/sort-ticket \
    -H "Content-Type: application/json" -d "$body"
  echo

curl -i -s -X POST http://localhost:8000/sort-ticket \
  -H "Content-Type: application/json" -d '{"message":"hello"}'
# -> HTTP/1.1 422 Unprocessable Content

curl -s -X POST http://localhost:8000/sort-ticket \
  -H "Content-Type: application/json" \

autocannon -c 50 -d 30 -j http://localhost:8000/health

  -H "Content-Type=application/json" \
  -b '{"ticket_id":"LOAD-1","message":"I sent 3000 to wrong number"}' \
  http://localhost:8000/sort-ticket

  -H "Content-Type=application/json" \
```
